# Functions
- Functions group reusable blocks of code.
- A function can be declared with the function keyword or created as an expression.
- Functions may accept parameters and may return values.
- A function without a return statement returns undefined.

# Parameters & Arguments
- Parameters act as placeholders for incoming values.
- Default parameters provide fallback values when no argument is passed.
- Functions can accept one or multiple parameters.

# Return Values
- return sends a value back to the caller.
- A function stops executing immediately after return.
- Early returns act like an implicit else.

# Arrow Functions
- Arrow functions provide a shorter syntax for writing functions.
- Implicit return works when the function body is a single expression.
- Returning an object requires wrapping it in parentheses ({ ... }).

# Function Expressions
- A function expression stores a function inside a variable.
- Can be anonymous or named.
- Useful for callbacks and passing functions as values.

# Scope
- Block scope: variables declared with let or const exist only inside {}.
- Function scope: variables declared inside a function are not accessible outside.
- Global scope: variables declared outside any block or function.
- Functions can mutate global variables unless shadowed by local ones.

# Deconstruction Drill (Concepts)
- A single function can be rewritten in multiple syntaxes:
- Function declaration
- Function expression
- Arrow function
- Explicit vs implicit return
- Functions can return objects, template literals, or primitive values.
- Validation inside functions prevents invalid input.
